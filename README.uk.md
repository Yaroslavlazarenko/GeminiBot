# Mia - Просунутий ШІ-Асистент для Telegram

Мія — це високотехнологічний, мультимодальний Telegram-бот з глибоко пропрацьованим характером, побудований на базі **Google Gemini API** (GenAI SDK) та **FastMCP**. Вона використовує сучасну архітектуру з MongoDB під капотом, має дворівневу систему фільтрації (Gatekeeper), ізоляцію «думок» (Structured Output), динамічну оркестрацію вбудованих інструментів та можливість безшовного підключення зовнішніх локальних MCP-мікросервісів.

## 🌟 Ключові можливості

- **Дворівнева архітектура (Gatekeeper Pattern):** 
  - Швидка та легка модель (`gemini-3.1-flash-lite`) працює як розумний фільтр. У групових чатах вона використовує багатокроковий ланцюжок міркувань (оцінюючи контекст та пошук по історії), щоб вирішити: відповісти, проігнорувати або тимчасово вимкнутися.
  - Основна модель (`gemini-3.5-flash`), яка відіграє характер, викликається тільки за необхідності, економлячи токени та захищаючи від спаму.
- **Структурована ізоляція думок (Захист від витоку промптів):**
  - При генерації відповіді Мія використовує **Structured Outputs (JSON Schema)**. Нейромережа зобов'язана розділяти свої внутрішні міркування, аналіз фото та чернетки у приховане поле `internal_monologue`. У Telegram йде тільки вичитане поле `message`, що повністю виключає ситуації, коли бот "думає вголос" у чаті та виходить з ролі.
- **Розумна пам'ять та контекст:**
  - **Вічна історія чатів:** Повні логи зберігаються у MongoDB. Мія може сама викликати інструменти для пошуку по минулим бесідам за ключовими словами або датами.
  - **Ізольована пам'ять про користувачів:** Мія активно спостерігає за співрозмовниками та назавжди запам'ятовує їхні факти, секрети та вподобання. Жорсткі програмні межі приватності гарантують, що факт, дізнаний в одному чаті, ніколи не спливе в іншому.
- **Просунута обробка медіа та стиснення:**
  - **Захист від спаму (Burst Debouncing):** Бот збирає швидкі потоки повідомлень та альбоми фотографій (із затримкою у 3 секунди), об'єднуючи їх в один мультимодальний промпт.
  - **Каскадне стиснення зображень:** При отриманні "важких" фотографій бот елегантно стискає їх каскадом (2K -> 1080p -> 720p), щоб відправити в LLM максимальну якість, не виходячи за ліміт контексту в 4.5 МБ.
  - **Кружечки та стікери:** Автоматично транскрибує аудіо через Groq Whisper і робить швидкий візуальний аналіз через Gemini Vision для відео-кружечків та Lottie/WebM стікерів, відправлених користувачем.
- **Розширюваність через MCP та Локальний файловий міст:**
  - **Вбудовані інструменти:** Мія автономно ставить реакції на повідомлення, відповідає реплаєм на конкретні цитати, шукає/відправляє потрібні стікери та записує **справжні голосові повідомлення** (через ElevenLabs TTS).
  - **Зв'язок через спільні Volume (Media Bridging):** Файли Telegram зберігаються тільки в RAM (пам'яті). Але якщо зовнішньому MCP-серверу (наприклад, Google Reverse Image Search) потрібен абсолютний шлях до локального файлу, Мія органічно знаходить інструмент `download_media_to_disk`, зберігає картинку у спільний Docker Volume (з автоочищенням старих файлів через 1 годину) і безшовно передає шлях зовнішньому мікросервісу.
  - **Віддалені MCP Сервери:** Підключення до зовнішніх HTTP/SSE MCP серверів прямо через Адмін-панель. Бот динамічно підтягує і оркеструє нові інструменти (наприклад, Exa Search) без втручання в код.
- **Бронебійне форматування:**
  - Реалізовано механізм Graceful Markdown/HTML Fallback. Якщо Telegram відхиляє форматування від ШІ (наприклад, через випадкові теги `<щось>`), бот перехоплює помилку, санітизує текст за допомогою `html.escape()` і перевідправляє його як звичайний текст. Бот ніколи не падає через кривий Markdown від нейромережі.

## Архітектура проєкту

1. **Docker-середовище:** Основний бот крутиться у контейнері Python 3.11. У сусідньому контейнері працює MongoDB.
2. **Спільний медіа-диск (Shared Volume):** Виділений Docker volume (`gemini_shared_media`) дозволяє боту безпечно складати медіафайли для сусідніх MCP-мостів (наприклад, інструментів на Node.js, що вимагають доступу до ФС).
3. **Web Адмін-Панель:** Веб-інтерфейс на базі FastAPI на порту `8081`. Доступ суворо захищений одноразовими токенами, які генеруються динамічно у самому Telegram власником бота. Дозволяє в реальному часі редагувати системні промпти, конфігурації MCP та API ключі без перезапуску бота.

## Швидкий старт (Docker)

Запуск здійснюється суворо через Docker та Docker Compose.

### 1. Клонування та налаштування
```bash
git clone https://github.com/Yaroslavlazarenko/GeminiBot.git
cd GeminiBot
cp .env.example .env
```

Відредагуйте файл `.env`, додавши базові ключі:
```env
BOT_TOKEN=your_telegram_bot_token
GEMINI_API_KEY=your_gemini_api_key
ADMIN_TELEGRAM_ID=your_telegram_user_id
```

### 2. Збірка та запуск
```bash
docker-compose up -d --build
```

### 3. Доступ в Адмін-панель
1. Відправте `/admin` вашому боту в Telegram.
2. Бот відповість захищеним одноразовим посиланням.
3. Перейдіть за посиланням, щоб налаштувати додаткові ключи (ElevenLabs, Groq), системні промпти, набори стікерів та ендпоінти зовнішніх MCP.

## Додавання зовнішніх MCP Серверів (SSE)

Ви можете нескінченно розширювати можливості Мії, взагалі не чіпаючи її основний код на Python.
1. Підніміть зовнішній MCP-сервер, що транслює дані у SSE (наприклад, використовуючи `@modelcontextprotocol/inspector` поверх звичайного `stdio` сервера) на тому ж хості.
2. Примонтуйте до нього volume `geminibot_gemini_shared_media`, якщо мікросервісу потрібен локальний доступ до файлів бота.
3. Відкрийте Адмін-панель Мії та додайте сервер у **MCP Config**:
```json
{
  "google_images": {
    "url": "http://mcp-images-standalone:5173/sse"
  },
  "exa_search": {
    "url": "https://mcp.exa.ai/mcp",
    "headers": {"Authorization": "Bearer YOUR_API_KEY"}
  }
}
```
Мія миттєво успадкує нові здібності на наступному ж ході.
