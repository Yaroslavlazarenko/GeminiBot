# Mia - Просунутий ШІ-асистент для Telegram

Mia — це високотехнологічний Telegram-бот із пропрацьованим характером, побудований на базі **Google Gemini API** (GenAI SDK) та **FastMCP**. Проєкт використовує сучасну чисту архітектуру зі сховищем MongoDB та включає двомодельну систему маршрутизації "Воротар" (Gatekeeper), динамічну оркестрацію інструментів і безпечну вебпанель адміністратора.

## 🌟 Основні можливості

- **Двомодельна архітектура (Gatekeeper Pattern):** 
  - Швидка та легка модель (`gemini-3.1-flash-lite`) аналізує кожне вхідне повідомлення, щоб визначити: це інформаційний шум, прохання замовкнути чи повідомлення, що потребує відповіді.
  - Основна модель персонажа (`gemini-3.5-flash`) викликається лише за необхідності, економлячи токени та запобігаючи галюцинаціям ШІ в активних групових чатах.
- **Нативна інтеграція FastMCP:**
  - **Локальні інструменти:** Mia нативно використовує інструменти для відправки реакцій у Telegram, відповідей на конкретні повідомлення, надсилання стікерів та генерації **справжніх голосових повідомлень** (через ElevenLabs TTS).
  - **Віддалені сервери MCP:** Бот легко підключається до зовнішніх серверів MCP за протоколами HTTP/SSE (наприклад, пошук в інтернеті, робота з файлами, калькулятори). Проксі-сервіс динамічно завантажує інструменти, вирішує конфлікти імен та автономно керує циклом `function_calling`.
- **Документо-орієнтоване сховище:** База даних MongoDB. Історії чатів та налаштування користувачів/груп зберігаються в чистих JSON-документах, які ідеально підходять під формат API Gemini. Жодних складних реляційних міграцій.
- **Безпечна вебпанель:** Дозволяє "на льоту" керувати моделями, перевизначати Gemini Base URL (для проксі), налаштовувати MCP сервери та редагувати системний промпт Мії. Доступ суворо захищений одноразовими токенами, що генеруються прямо в Telegram.
- **100% Docker:** Жодного ручного налаштування середовища чи системних скриптів не потрібно.

## 🚀 Швидкий старт (Docker)

Розгортання здійснюється виключно через Docker та Docker Compose.

### 1. Вимоги
- Встановлені Docker та Docker Compose на вашому сервері.
- Токен Telegram бота (від [@BotFather](https://t.me/BotFather)).
- Ключ Google Gemini API (отримати в [Google AI Studio](https://aistudio.google.com/)).
- Ваш Telegram User ID (для доступу до адмін-панелі).

### 2. Встановлення

Склонуйте репозиторій:
```bash
git clone https://github.com/Yaroslavlazarenko/GeminiBot.git
cd GeminiBot
```

Налаштуйте змінні середовища:
```bash
cp .env.example .env
nano .env
```

**Обов'язкові поля в `.env`:**
```env
BOT_TOKEN=1234567890:ВАШ_ТОКЕН_TELEGRAM_БОТА
GEMINI_API_KEY=ВАШ_КЛЮЧ_GEMINI_API
ADMIN_TELEGRAM_ID=ВАШ_ОСОБИСТИЙ_ID_TELEGRAM # наприклад, 123456789

# Необов'язково: Для генерації голосових повідомлень
ELEVENLABS_API_KEY=ваш_ключ_elevenlabs
```

### 3. Запуск бота
```bash
docker compose up -d --build
```
Будуть запущені два контейнери: `mongodb` та сам `bot`.

## 🔐 Вебпанель адміністратора

Mia містить вбудований вебінтерфейс (за замовчуванням на порту `8081`) для динамічного керування ботом без перезапуску Docker.

### Як отримати доступ:
1. Переконайтеся, що ваш `ADMIN_TELEGRAM_ID` вказано у файлі `.env`.
2. Надішліть боту в Telegram команду `/admin`.
3. Бот згенерує безпечне одноразове посилання:
   `http://<IP_ВАШОГО_СЕРВЕРА>:8081/?token=СЕКРЕТНИЙ_ХЕШ`
4. Перейдіть за посиланням. Браузер безпечно авторизує вас через HTTP cookies.

### Що можна налаштувати:
- **System Instruction:** Змінити характер Мії, її передісторію та правила поведінки.
- **Models:** Змінити модель Гейткіпера або Основну модель "на льоту".
- **Gemini Base URL:** Перевизначити стандартний URL Google API (наприклад, для скерування трафіку через кастомний проксі).
- **MCP Servers Config:** Підключити віддалені сервери MCP за допомогою JSON-конфігурації.

#### Приклад конфігурації серверів MCP:
```json
{
  "math_server": {
    "url": "https://mathematics.fastmcp.app/mcp"
  },
  "search_server": {
    "url": "https://mcp.exa.ai/mcp",
    "type": "sse",
    "headers": {
      "Authorization": "Bearer ВАШ_API_КЛЮЧ"
    }
  }
}
```
*Після натискання "Save" в панелі, бот динамічно перепідключиться до нових серверів MCP, завантажить їхні схеми та впровадить їх у "мозок" Мії, не пропустивши при цьому жодного повідомлення в Telegram.*

## 🧠 Архітектура

Проєкт дотримується принципів Прагматичної Чистої Архітектури (Pragmatic Clean Architecture):
- **`core/`**: Централізовані налаштування, підключення до БД (`motor`), перерахування (Enums) та логування.
- **`services/`**:
  - `ai_service.py`: Оркестрація основної моделі Gemini та автономного циклу виконання інструментів MCP.
  - `gatekeeper_service.py`: Оцінює вхідний контекст і приймає рішення: `RESPOND` (Відповісти), `IGNORE` (Ігнорувати) або `DISABLE_RESPONSES` (Замовкнути).
  - `mcp_manager.py`: Обробка динамічних HTTP/SSE підключень до віддалених MCP-серверів і вирішення конфліктів імен інструментів.
  - `tts_service.py`: Генерація аудіофайлів через ElevenLabs.
- **`bot/`**: Шар представлення Telegram. `handlers.py` зіставляє події Telegram з уніфікованою абстракцією `ChatContext`.
