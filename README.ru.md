# Mia - Продвинутый ИИ-Ассистент для Telegram

Мия — это высокотехнологичный, мультимодальный Telegram-бот с проработанным характером, построенный на базе **Google Gemini API** (GenAI SDK) и **FastMCP**. Созданная для того, чтобы ощущаться как реальный собеседник, Мия обладает долговременной памятью, умеет анализировать фотографии, смотреть "кружочки", слушать голосовые сообщения и отвечать настоящим аудио-голосом.

Под капотом Мия использует современную контейнеризированную архитектуру с базой данных MongoDB, двухуровневую систему маршрутизации запросов ("Gatekeeper"), изоляцию мыслей через структурированный вывод (Structured Output) и бесшовное подключение внешних микросервисов по протоколу MCP (Model Context Protocol).

---

## Ключевые возможности

### Двухуровневый Gatekeeper (Оптимизация для групп)
Запускать ИИ в активном групповом чате может быть дорого и спамно. Мия решает эту проблему с помощью двухуровневой системы:
- **Gatekeeper (`gemini-3.1-flash-lite`):** Быстрая и дешевая модель, которая читает каждое входящее сообщение в группе. Она оценивает контекст беседы, при необходимости ищет информацию в истории чата и решает: должна ли Мия *Ответить*, *Проигнорировать* или *Временно отключиться*.
- **Основа характера (`gemini-3.5-flash`):** Более тяжелая и умная модель, которая генерирует реальные ответы Мии только тогда, когда Gatekeeper одобряет вмешательство.

### Изоляция мыслей (Защита от утечки промптов)
Мия генерирует ответы используя **Structured Outputs (JSON Schema)**. Нейросеть обязана разделять свои внутренние рассуждения, анализ фото и черновики в скрытое поле `internal_monologue`. В Telegram отправляется только вычитанное поле `message`. Это полностью исключает ситуации, когда бот случайно отправляет в чат маркеры вроде `/thought` или свои аналитические размышления.

### Мультимодальность: Зрение, Голос и Видео
- **Кружочки и Голосовые:** Автоматически транскрибирует аудио через ультрабыстрый **Groq Whisper API**. Видео-кружочки также проходят через Gemini Vision для понимания визуального контекста.
- **Каскадное сжатие фото:** При получении "тяжелых" фотографий бот плавно сжимает их каскадом (2K -> 1080p -> 720p). Это гарантирует, что в LLM отправится максимально возможное качество без превышения лимита контекста в 4.5 МБ.
- **Каталог стикеров:** Фоновые процессы скачивают настроенные вами паки стикеров Telegram, прогоняют их через Gemini Vision для генерации описаний эмоций и сохраняют в базу. Мия использует локальный инструмент, чтобы выбирать и отправлять идеальный стикер.
- **Настоящие голосовые ответы:** Мия умеет записывать голосовые сообщения используя **ElevenLabs TTS API**.

### Постоянная память и контекст
- **Вечная история:** Полные логи чатов хранятся в MongoDB. Мия может сама вызывать инструменты для поиска по прошлым беседам по ключевым словам или датам.
- **Изолированная память о пользователях:** Мия активно наблюдает за собеседниками и навсегда запоминает их факты, секреты и предпочтения. Жесткие программные границы приватности гарантируют, что чувствительный факт, узнанный в личном чате, никогда не всплывет в публичной группе.
- **Понимание контекста:** Бот перехватывает нативные свойства Telegram (`reply` и `forward`), добавляя точную атрибуцию того, кто и что сказал, чтобы ИИ идеально понимал сложные ветки диалогов.

### Расширяемость через MCP и Локальный медиа-мост
Расширяйте возможности Мии бесконечно, не меняя её основной код:
- **Локальные инструменты:** Мия автономно ставит реакции, отвечает реплаем на конкретные цитаты, ищет стикеры и сохраняет факты в память.
- **Общий диск (Media Bridging):** Файлы Telegram существуют только в оперативной памяти. Если внешнему MCP-серверу (например, Google Reverse Image Search) нужен абсолютный путь к файлу на диске, Мия органично находит инструмент `download_media_to_disk`, сохраняет картинку в общий Docker Volume (с автоочисткой старых файлов через 1 час) и бесшовно передает путь микросервису.
- **Удаленные MCP Серверы:** Подключайте внешние HTTP/SSE MCP серверы через Админ-панель. Бот динамически подтягивает инструменты (например, Exa Search) и оркестрирует их автономно.

## Архитектура проекта

1. **Docker-окружение:** Основной бот крутится в контейнере Python 3.11. В соседнем контейнере работает MongoDB.
2. **Общий медиа-диск (Shared Volume):** Выделенный Docker volume (`gemini_shared_media`) позволяет боту безопасно складывать медиафайлы для соседних MCP-мостов (например, инструментов на Node.js, требующих доступ к ФС).
3. **Web Админ-Панель:** Веб-интерфейс на базе FastAPI на порту `8081`. Доступ строго защищен одноразовыми токенами, которые генерируются динамически в самом Telegram владельцем бота. Позволяет в реальном времени редактировать системные промпты, конфигурации MCP и API ключи без перезапуска бота.

## Требования

- **Docker** и **Docker Compose** установленные на вашем сервере.
- Токен Telegram бота от [@BotFather](https://t.me/BotFather).
- Ключ Google Gemini API от [Google AI Studio](https://aistudio.google.com/).
- *(Опционально)* Ключ Groq API для быстрой транскрипции голоса.
- *(Опционально)* Ключ ElevenLabs API для генерации голосовых ответов.

## Установка и быстрый старт

### 1. Клонирование и настройка
```bash
git clone https://github.com/Yaroslavlazarenko/GeminiBot.git
cd GeminiBot
cp .env.example .env
```

Отредактируйте файл `.env`, добавив ваши ключи:
```env
BOT_TOKEN=your_telegram_bot_token
GEMINI_API_KEY=your_gemini_api_key
ADMIN_TELEGRAM_ID=your_telegram_user_id # Важно для доступа к Админ-панели
# Опционально:
GROQ_API_KEY=your_groq_api_key
ELEVENLABS_API_KEY=your_elevenlabs_api_key
```

### 2. Сборка и запуск
```bash
docker-compose up -d --build
```
Бот запустится, подключится к MongoDB, создаст индексы и начнет работать.

### 3. Доступ в безопасную Админ-панель
1. Отправьте команду `/admin` вашему боту в Telegram.
2. Если ваш Telegram ID совпадает с `ADMIN_TELEGRAM_ID`, бот ответит одноразовой ссылкой.
3. Перейдите по ссылке в браузере, чтобы настроить:
   - Системный промпт (личность Мии).
   - Дополнительные API ключи.
   - Паки стикеров.
   - Эндпоинты внешних MCP серверов.

## Как общаться с Мией

- **Личные чаты:** Просто отправьте ей сообщение, фото или голосовое. Мия собирает быстрые потоки сообщений (отправленные с задержкой менее 3 секунд) в один контекст, чтобы не спамить вас ответами на каждое слово.
- **Групповые чаты:** Добавьте Мию в группу. **Важно:** Чтобы Мия видела все сообщения (и Gatekeeper мог анализировать контекст), вы должны либо сделать её Администратором, либо отключить Group Privacy Mode через @BotFather.
- **Команды:**
  - `/clear` - Очищает краткосрочное окно контекста бота (но сохраняет долговременную память в БД).
  - `/admin` - Генерирует ссылку для входа в Админ-панель (только для владельца).

## Добавление внешних MCP Серверов (SSE)

Вы можете подключать сторонние MCP-серверы (веб-поиск, интеграция с GitHub или реверсивный поиск по картинкам) без изменения кода бота.

1. Поднимите внешний MCP-сервер, транслирующий данные в SSE (например, используя `@modelcontextprotocol/inspector` поверх обычного `stdio` сервера) на том же хосте.
2. Если MCP-серверу нужен доступ к картинкам из Telegram, обязательно примонтируйте к нему volume `geminibot_gemini_shared_media` в папку `/tmp/gemini_media`.
3. Откройте Админ-панель Мии и добавьте сервер в **MCP Config**:
```json
{
  "google_images": {
    "url": "http://mcp-images-standalone:5173/sse"
  },
  "exa_search": {
    "url": "https://mcp.exa.ai/mcp",
    "headers": {"Authorization": "Bearer YOUR_EXA_API_KEY"}
  }
}
```
Мия мгновенно унаследует новые способности на следующем же ходе диалога.
