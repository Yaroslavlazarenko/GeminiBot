# Mia - Продвинутый ИИ-ассистент для Telegram

Mia — это высокотехнологичный Telegram-бот с проработанным характером, построенный на базе **Google Gemini API** (GenAI SDK) и **FastMCP**. Проект использует современную чистую архитектуру с хранилищем MongoDB и включает двухмодельную систему маршрутизации "Привратник" (Gatekeeper), динамическую оркестрацию инструментов и безопасную веб-панель администратора.

## 🌟 Основные возможности

- **Двухмодельная архитектура (Gatekeeper):** 
  - Быстрая и легкая модель (`gemini-2.5-flash-8b`) анализирует каждое входящее сообщение, чтобы определить: это информационный шум, просьба замолчать или сообщение, требующее ответа.
  - Основная модель персонажа (`gemini-2.5-flash`) вызывается только при необходимости, экономя токены и предотвращая галлюцинации ИИ в активных групповых чатах.
- **Нативная интеграция FastMCP:**
  - **Локальные инструменты:** Mia нативно использует инструменты для отправки реакций в Telegram, ответов на конкретные сообщения, отправки стикеров и генерации **настоящих голосовых сообщений** (через ElevenLabs TTS).
  - **Удаленные MCP серверы:** Бот легко подключается к внешним серверам MCP по протоколам HTTP/SSE (например, поиск в интернете, работа с файлами, калькуляторы). Прокси-сервис динамически подгружает инструменты, разрешает конфликты имен и автономно управляет циклом `function_calling`.
- **Документо-ориентированное хранилище:** База данных MongoDB. Истории чатов и настройки пользователей/групп хранятся в чистых JSON-документах, которые идеально подходят под формат API Gemini. Никаких сложных реляционных миграций.
- **Безопасная Веб-Панель:** Позволяет "на лету" управлять моделями, переопределять Gemini Base URL (для прокси), настраивать MCP серверы и редактировать системный промпт Мии. Доступ строго защищен одноразовыми токенами, генерируемыми прямо в Telegram.
- **100% Docker:** Никакой ручной настройки окружения или системных скриптов не требуется.

## 🚀 Быстрый старт (Docker)

Развертывание осуществляется исключительно через Docker и Docker Compose.

### 1. Требования
- Установленные Docker и Docker Compose на вашем сервере.
- Токен Telegram бота (от [@BotFather](https://t.me/BotFather)).
- Ключ Google Gemini API (получить в [Google AI Studio](https://aistudio.google.com/)).
- Ваш Telegram User ID (для доступа к админ-панели).

### 2. Установка

Склонируйте репозиторий:
```bash
git clone https://github.com/Yaroslavlazarenko/GeminiBot.git
cd GeminiBot
```

Настройте переменные окружения:
```bash
cp .env.example .env
nano .env
```

**Обязательные поля в `.env`:**
```env
BOT_TOKEN=1234567890:ВАШ_ТОКЕН_TELEGRAM_БОТА
GEMINI_API_KEY=ВАШ_КЛЮЧ_GEMINI_API
ADMIN_TELEGRAM_ID=ВАШ_ЛИЧНЫЙ_ID_TELEGRAM # например, 123456789

# Необязательно: Для генерации голосовых сообщений
ELEVENLABS_API_KEY=ваш_ключ_elevenlabs
```

### 3. Запуск бота
```bash
docker compose up -d --build
```
Будут запущены два контейнера: `mongodb` и сам `bot`.

## 🔐 Веб-панель администратора

Mia включает встроенный веб-интерфейс (по умолчанию на порту `8081`) для динамического управления ботом без перезапуска Docker.

### Как получить доступ:
1. Убедитесь, что ваш `ADMIN_TELEGRAM_ID` указан в файле `.env`.
2. Отправьте боту в Telegram команду `/admin`.
3. Бот сгенерирует безопасную одноразовую ссылку:
   `http://<IP_ВАШЕГО_СЕРВЕРА>:8081/?token=СЕКРЕТНЫЙ_ХЕШ`
4. Перейдите по ссылке. Браузер безопасно авторизует вас через HTTP cookies.

### Что можно настроить:
- **System Instruction:** Изменить характер Мии, её предысторию и правила поведения.
- **Models:** Поменять модель Гейткипера или Основную модель "на лету".
- **Gemini Base URL:** Переопределить стандартный URL Google API (например, для направления трафика через кастомный прокси).
- **MCP Servers Config:** Подключить удаленные серверы MCP с помощью JSON-конфигурации.

#### Пример конфигурации серверов MCP:
```json
{
  "math_server": {
    "url": "https://mathematics.fastmcp.app/mcp"
  },
  "search_server": {
    "url": "https://mcp.exa.ai/mcp",
    "type": "sse",
    "headers": {
      "Authorization": "Bearer ВАШ_API_КЛЮЧ"
    }
  }
}
```
*При нажатии "Save" в панели, бот динамически переподключится к новым серверам MCP, скачает их схемы и внедрит их в "мозг" Мии, не пропустив при этом ни одного сообщения в Telegram.*

## 🧠 Архитектура

Проект следует принципам Прагматичной Чистой Архитектуры (Pragmatic Clean Architecture):
- **`core/`**: Централизованные настройки, подключение к БД (`motor`), перечисления (Enums) и логирование.
- **`services/`**:
  - `ai_service.py`: Оркестрация основной модели Gemini и автономного цикла выполнения инструментов MCP.
  - `gatekeeper_service.py`: Оценивает входящий контекст и принимает решения: `RESPOND` (Ответить), `IGNORE` (Игнорировать) или `DISABLE_RESPONSES` (Замолчать).
  - `mcp_manager.py`: Обработка динамических HTTP/SSE подключений к удаленным MCP-серверам и разрешение конфликтов имен инструментов.
  - `tts_service.py`: Генерация аудиофайлов через ElevenLabs.
- **`bot/`**: Слой представления Telegram. `handlers.py` сопоставляет события Telegram с унифицированной абстракцией `ChatContext`.
